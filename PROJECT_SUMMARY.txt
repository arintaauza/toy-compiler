LANGUAGE FEATURES:
═══════════════════════════════════════════════════════════════════════════════

✅ Static typing (int, float, bool, string, void)
✅ Variables and constants (let, const)
✅ Functions with parameters and return types
✅ Control flow (if/else, while loops)
✅ Expressions with operators (+, -, *, /, %, ==, !=, <, >, <=, >=, &&, ||, !)
✅ Built-in functions (print, input, len)
✅ C-like syntax (fn, let, return, {})
✅ Comments (// single-line, /* multi-line */)

Example Luna Program:
```luna
fn fibonacci(n: int) -> int {
    if n <= 1 {
        return n;
    }
    return fibonacci(n - 1) + fibonacci(n - 2);
}

fn main() -> int {
    let result: int = fibonacci(10);
    print(result);  // Prints: 55
    return 0;
}
```

COMPILER ARCHITECTURE (8 Phases - Production Quality):
═══════════════════════════════════════════════════════════════════════════════

Source Code (.luna)
    ↓
[Phase 1] Lexical Analysis (Lexer)
    → Tokens
    ↓
[Phase 2] Syntax Analysis (Parser)
    → Abstract Syntax Tree (AST)
    ↓
[Phase 3] Semantic Analysis
    → Type-checked AST + Symbol Table
    ↓
[Phase 4] Intermediate Representation
    → Three-Address Code (TAC) + Control Flow Graph (CFG)
    ↓
[Phase 5] IR Optimizations
    → Optimized IR (DCE, CSE, Constant Folding, Algebraic, Copy Propagation)
    ↓
[Phase 6] Assembly Code Generation
    → x86-64 or ARM64 Assembly (with Register Allocation)
    ↓
[Phase 7] LLVM IR Generation (Optional)
    → LLVM IR → Optimized with LLVM passes → Native Code
    ↓
[Phase 8] Production Polish
    → Professional CLI, Benchmarks, Documentation

LEARNING OBJECTIVES (Production Compiler):
═══════════════════════════════════════════════════════════════════════════════

By building this compiler, you'll learn:

**Compiler Frontend:**
1. Lexical Analysis: Regular expressions, pattern matching, tokenization
2. Parsing: Context-free grammars, recursive descent, operator precedence
3. Type Systems: Static typing, type checking, type inference basics
4. Symbol Tables: Scope management, name resolution
5. Semantic Analysis: AST traversal, validation, error detection

**Intermediate Representation:**
6. Three-Address Code (TAC): IR design, instruction sets
7. Control Flow Graphs (CFG): Basic blocks, dominators, data flow
8. SSA Form: Static single assignment (in LLVM phase)

**Optimizations:**
9. Dataflow Analysis: Liveness analysis, reaching definitions
10. Constant Folding: Compile-time evaluation
11. Dead Code Elimination: Liveness-based removal
12. Common Subexpression Elimination: Value numbering
13. Copy Propagation: Dataflow-based substitution
14. Algebraic Simplification: Mathematical identities, strength reduction
15. Branch Elimination: Control flow simplification
16. Pass Management: Fixed-point iteration, pass ordering

**Compiler Backend:**
17. Register Allocation: Linear scan, graph coloring concepts
18. Stack Frame Layout: Calling conventions, ABI compliance
19. x86-64 Assembly: Instruction selection, code emission
20. System V ABI: Function calling, parameter passing
21. Linking: Object files, symbol resolution, C library integration

**LLVM (Optional):**
22. LLVM IR: SSA-based IR, type system, instruction set
23. LLVM API: Module building, IR generation, verification
24. LLVM Optimizations: -O3 pipeline, inlining, vectorization
25. LLVM Compilation: llc, clang integration, multiple targets

**Software Engineering:**
26. Software Architecture: Multi-phase design, separation of concerns
27. Testing: TDD, unit tests, integration tests, correctness validation
28. Error Handling: Meaningful messages, error recovery, diagnostics
29. Performance: Benchmarking, profiling, optimization impact analysis
30. Documentation: Technical writing, API docs, tutorials
